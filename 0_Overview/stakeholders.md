# ステークホルダー

## ステークホルダー一覧

| 役割 | 名前 | 責務 | 意思決定権限 | 連絡先 |
|------|------|------|--------------|--------|
| プロジェクトマネージャー | TBD | プロジェクト全体の管理・調整 | 高 | TBD |
| プロダクトオーナー | TBD | 要件定義・優先順位決定 | 高 | TBD |
| テックリード | TBD | 技術的決定・アーキテクチャ | 高 | TBD |
| フロントエンドリード | TBD | フロントエンド開発 | 中 | TBD |
| バックエンドリード | TBD | バックエンド開発 | 中 | TBD |
| デザイナー | TBD | UI/UX設計 | 中 | TBD |
| QAリード | TBD | 品質保証・テスト | 中 | TBD |

## RACIマトリックス

### 主要プロセス

| プロセス | プロジェクトマネージャー | プロダクトオーナー | テックリード | フロントエンドリード | バックエンドリード | デザイナー | QAリード |
|----------|-------------------------|-------------------|--------------|---------------------|-------------------|------------|----------|
| 要件定義 | A | R | C | C | C | C | I |
| 設計 | A | C | R | C | C | C | I |
| 開発 | A | C | C | R | R | C | I |
| テスト | A | C | C | C | C | I | R |
| リリース | R | A | C | C | C | I | C |

**凡例**: R = Responsible（実行責任）, A = Accountable（説明責任）, C = Consulted（相談先）, I = Informed（報告先）

## 意思決定フロー

1. **要件レベルの決定**: プロダクトオーナーが最終決定
2. **技術レベルの決定**: テックリードが最終決定（重大な場合はプロジェクトマネージャーと協議）
3. **デザインレベルの決定**: デザイナーが最終決定（プロダクトオーナーの承認が必要）
